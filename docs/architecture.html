The Cube Model

El Modelo del Cubo

Rubik OS organizes the operating system as a Rubik's Cube: 6 faces with 58 atomic cells. Each cell is an independent process with its own cgroup, AppArmor profile, and resource limits.

Rubik OS organiza el sistema operativo como un Cubo Rubik: 6 caras con 58 celdas atómicas. Cada celda es un proceso independiente con su propio cgroup, perfil AppArmor y límites de recursos.

Principles

Principios

DecentralizationDescentralización — Each cell is independent. If one fails, others keep running.Cada celda es independiente. Si una falla, las demás continúan.

AtomicityAtomicidad — Each cell does exactly one thing.Cada celda hace exactamente una cosa.

Hot-swappableIntercambiabilidad — Any cell can be replaced at runtime.Cada celda puede ser reemplazada en caliente.

Memory EfficiencyEficiencia — ZRAM compression, on-demand services, shared memory.Compresión ZRAM, servicios bajo demanda, memoria compartida.

Flat HierarchyJerarquía plana — No unnecessary abstraction layers. Cells communicate directly.Sin capas de abstracción innecesarias. Comunicación directa.

Orchestrator (rubikd)

Orquestador (rubikd)

Manages cell startup/shutdown, dependencies, rotation, health checks, and memory metrics.

Gestiona inicio/parada de celdas, dependencias, rotación, health checks y métricas de memoria.

rubikctl status          # Status of all cells# Estado de todas las celdas
rubikctl rotate face-0   # Replace all cells in a face# Reemplazar todas las celdas de una cara
rubikctl cell enable F0.4 memory-compression
rubikctl cell disable F2.2 defrag

Cell Lifecycle

Ciclo de Vida

REGISTERED → LOADED → ACTIVE → HEALTHY
                          ↓
                     DEGRADED → RETRY → ACTIVE
                          ↓
                      FAILED → NOTIFY → RESTART

The orchestrator monitors each cell's state and executes corrective actions automatically.

El orquestador monitorea el estado de cada celda y ejecuta acciones correctivas automáticamente.
