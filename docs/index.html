About Rubik OS

Acerca de Rubik OS

Rubik OS is an Arch-based Linux distribution that runs on 80-120 MB RAM at idle. It reorganizes every component as an independent cell — like the faces of a Rubik's Cube — with its own cgroup, AppArmor profile, and resource limits.

Rubik OS es una distribución Linux basada en Arch que funciona con 80-120 MB RAM en idle. Reorganiza cada componente como una celda independiente — como las caras de un Cubo Rubik — con su propio cgroup, perfil AppArmor y límites de recursos.

The Rubik's Cube Model

El Modelo del Cubo Rubik

Rubik OS organizes the operating system as a Rubik's Cube: 6 faces, each with 9 cells (for F0-F4) or 11 cells (F3), totaling 58 atomic cells.

Rubik OS organiza el sistema operativo como un Cubo Rubik: 6 caras, cada una con 9 celdas (F0-F4) u 11 celdas (F3), totalizando 58 celdas atómicas.

Architectural Principles

Principios Arquitectónicos

  • DecentralizationDescentralización — Each cell is an independent process. If one fails, others keep running.Cada celda es un proceso independiente. Si una falla, las demás continúan.
  • AtomicityAtomicidad — Each cell does exactly one thing.Cada celda hace exactamente una cosa.
  • Hot-swappable (Rotation)Intercambiabilidad (Rotación) — Any cell can be replaced at runtime.Cada celda puede ser reemplazada en caliente.
  • Memory EfficiencyEficiencia de Memoria — ZRAM compression, on-demand services, shared memory pool between face cells.Compresión ZRAM, servicios bajo demanda, pool de memoria compartida entre celdas de la misma cara.
  • Flat HierarchyJerarquía plana — No unnecessary abstraction layers. Cells communicate directly.Sin capas de abstracción innecesarias. Comunicación directa entre celdas.

The Rubik Orchestrator (rubikd)

El Orquestador Rubik (rubikd)

The central orchestrator manages cell lifecycle, dependencies, health checks, and memory metrics.

El orquestador central gestiona el ciclo de vida de las celdas, dependencias, health checks y métricas de memoria.

rubikctl status          # Status of all cells# Estado de todas las celdas
rubikctl rotate face-0   # Replace all cells in a face# Reemplazar todas las celdas de una cara
rubikctl cell enable F0.4 memory-compression
rubikctl cell disable F2.2 defrag

Package Repository [rubik]

Repositorio de paquetes [rubik]

Add the official Rubik OS repo to your /etc/pacman.conf:

Agregá el repo oficial de Rubik OS a tu /etc/pacman.conf:

[rubik]
Server = https://erac73.github.io/rubik-os/repo/$arch

Available packages:

Paquetes disponibles:

  • rubik-core — Rubik OS base (orchestrator, cells, faces, systemd units, AppArmor profiles)Sistema base de Rubik OS (orquestador, celdas, caras, systemd units, perfiles AppArmor)
  • rubik-face-0 to rubik-face-5 — Scripts for each Rubik's Cube faceScripts de cada cara del Cubo Rubik

Then install with pacman:

Luego instalá con pacman:

pacman -Sy rubik-core

Cell Lifecycle

Ciclo de Vida de una Celda

REGISTERED → LOADED → ACTIVE → HEALTHY
                          ↓
                     DEGRADED → RETRY → ACTIVE
                          ↓
                      FAILED → NOTIFY → RESTART

The orchestrator monitors each cell's state and executes corrective actions automatically.

El orquestador monitorea el estado de cada celda y ejecuta acciones correctivas automáticamente.
